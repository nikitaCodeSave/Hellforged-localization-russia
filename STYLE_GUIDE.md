# Hellforged — Руководство по стилю RU-локализации

Версия: v1 (черновой эталон). Жанр: мрачное фэнтези, roguelite-экспедиции. Школа перевода — Diablo / Path of Exile / Last Epoch / Hades. Этот документ — обязательный эталон для всех команд. При конфликте с личным вкусом переводчика — приоритет у руководства.

---

## 0. Золотые правила (TL;DR)

1. **Безличность в UI и тултипах**, живой характер в диалогах.
2. **Терминология — только из мастер-глоссария.** Один EN-термин = один RU-термин по всей игре.
3. **Без английского Title Case.** В русских названиях с заглавной — только первое слово и имена собственные.
4. **Кавычки — ёлочки «».** Тире — длинное «—».
5. **Плейсхолдеры `{0}`, `{minValue}`, разметку `<1>…</1>`, `<color=…>`, `<sprite name="…">` НЕ трогать, НЕ переводить, НЕ менять порядок токенов.**
6. **Числа форматирует движок.** Переводим текст вокруг числа, само число оставляем плейсхолдером.
7. Короткие UI-ярлыки — **без точки** в конце.

---

## 1. Форма обращения к игроку

Hellforged смешивает сухой UI и атмосферные реплики. Правило зависит от домена:

### 1.1. UI, тултипы характеристик, описания эффектов — БЕЗЛИЧНО
Не «вы», не «ты». Используем безличные/инфинитивные конструкции и описание свойства, а не действия игрока.

- Кнопки-действия: инфинитив или отглагольное существительное.
  `Equip` → **Экипировать**; `Salvage` → **Разобрать**; `Infuse` → **Влить**; `Ascend` → **Вознести**.
- Описания статов: безличное описание свойства.
  `Increases critical chance` → **Повышает шанс крита** (НЕ «вы повышаете»).
  `A chance to deal a Critical Damage` → **Шанс нанести критический урон**.
- Системные подсказки/онбординг: допустима форма «вы» там, где без неё фраза ломается, но **по умолчанию безлично**.
  `Open inventory and equip it to get stronger` → **Откройте инвентарь и наденьте предмет, чтобы стать сильнее** (онбординг — обращение на «вы», т.к. это прямой обучающий адрес).

**Развилка решена так:** холодный системный слой (статы, ярлыки, кнопки) — безлично; обучающие/направляющие подсказки (ONBOARDING_*, предупреждения) — на «вы». Никогда не «ты» в UI.

### 1.2. Боевые реплики игрока (HERO_*_BARK) — от первого лица
Это внутренний голос героя. Первое лицо, ед. число, по характеру (Торен — суровый паладин; Мортензия — холодная высокомерная).
`I will not kneel.` → **Я не преклоню колен.**

### 1.3. Диалоги NPC — по характеру персонажа, на «ты»
NPC обращаются к герою на **«ты»** (лагерь выживших, фамильярность, грубоватость). Тон — по персонажу (см. глоссарий, раздел NPC):
- **Гаррик** (летописец) — сухой, саркастичный, заботливый под маской.
- **Келдор** (кузнец) — резкий, немногословный, приземлённый.
- **Тень** (мистик) — загадочный, торжественно-зловещий, на грани архаики.
- **Мортензия** — высокомерная, ледяная.
`Still breathing? Good. Keep doing that.` → **Ещё дышишь? Хорошо. Так и продолжай.**

## 2. Регистр и тон по доменам

| Домен | Регистр/тон | Заглавные | Точка в конце |
|---|---|---|---|
| Имена снаряжения (gear) | Прил.+сущ., мрачное фэнтези | Только первое слово | Нет |
| Способности (NAME) | Ёмко, эффектно, как в скиллбаре | Только первое слово | Нет |
| Описания способностей (DESC) | Нейтрально-боевое, безлично | Обычное предложение | Да, если предложение |
| Характеристики/статусы | Сухо, точно, как в тултипе | Только первое слово | Лейбл — нет; описание — да |
| Достижения | Можно с искрой/каламбуром (но без отсебятины) | Только первое слово | Описание — да |
| Лор/диалоги | Литературный, атмосферный, живой | По правилам прозы | Да |
| UI-меню/настройки | Нейтрально-индустриальный (как в AAA) | Только первое слово | Нет |
| Системные предупреждения | Чёткий, прямой, на «вы» | Обычное предложение | Да |

### 3.1. Базовые кнопки

| Cancel | Отмена |
| Close | Закрыть |
| Accept | Принять |
| Confirm | Подтвердить |
| Delete | Удалить |
| Create | Создать |
| Resume | Продолжить |
| Restart | Заново |
| Quit | Выход |
| Main Menu | Главное меню |
| Give Up | Сдаться |
| Continue | Продолжить |

### 3.2. Аббревиатуры и единицы
- **DPS** → оставляем **DPS** (устоялось; не «УВС»).
- **SFX** → **Звук. эффекты** (в настройках); **VSync** → **Верт. синхр.** или **VSync** (оставить латиницей допустимо); **SSAO**, **Anti-Aliasing** → **Сглаживание**.
- Секунды: `{0}s` → **{0} с** (пробел + «с», без точки). `{0}m` (радиус/дистанция) → **{0} м**.
- Проценты: число несёт движок (`{0}` уже содержит «%»), поэтому НЕ добавляем «%» руками, если он внутри плейсхолдера. Если в EN явный литеральный `%` вне плейсхолдера — переносим как есть: «**на 5%**» (без пробела перед знаком, по типографике РФ — слитно: «5%»).
- Плюс к стату: `+{0} Armor` → **+{0} к броне** (предлог «к» + дательный падеж) ИЛИ в сухих списках нод — **+{0} брони**. Внутри одного домена — единообразно (в Вечном Древе принят формат «**+{0} к <стату>**»).

## 4. Пунктуация и типографика

1. **Кавычки — ёлочки «».** Вложенные — „лапки". Прямые `"` и `'` в финальном тексте недопустимы (кроме как внутри кода/плейсхолдеров).
2. **Тире — длинное «—»** с пробелами вокруг. Дефис «-» только внутри слов. В EN встречается `‑` (узкий дефис, напр. «Battle‑Scarred») — в RU превращаем в обычное слово без дефиса либо стандартный дефис по правилам.
3. **Короткие UI-ярлыки (кнопки, лейблы, названия) — без точки.** Многострочные/предложения в тултипах и диалогах — с точкой.
4. **Заглавные буквы — НЕ как в английском.** EN пишет «Critical Damage Bonus» (Title Case) — это особенность английского UI, НЕ перенос. В RU: **«Бонус критического урона»** (заглавная только в начале). Имена собственные и названия систем — с заглавной (см. глоссарий): Вечное Древо, Обелиск, Перегрузка, Арсенал.
5. **Многоточие — символ «…»**, не три точки.
6. **`\n` (перенос строки) сохраняем как есть** — это разметка, не текст.
7. Названия систем-сущностей внутри предложений пишем с заглавной как термин: «Запечатывайте Разломы, пока открыт Обелиск».

## 5. Множественное число и SmartFormat plural

Движок использует SmartFormat. Конструкция `{amount:plural:форма1|форма2|форма3}` в русском требует **трёх форм** (а не двух, как в EN).

Русские формы по правилу:
- **форма1** — для чисел, оканчивающихся на 1, кроме 11 (1 заряд, 21 заряд).
- **форма2** — для 2–4, кроме 12–14 (2 заряда, 3 заряда).
- **форма3** — для 0, 5–20, и оканчивающихся на 0/5–9 (5 зарядов, 11 зарядов, 0 зарядов).

Шаблон: `{n:plural:заряд|заряда|зарядов}`.

Если в исходнике только 2 формы (`{n:plural:second|seconds}` → EN one/other), в RU **обязательно расширяем до 3**:
`{seconds:plural:секунду|секунды|секунд}`.

Где число не склоняемо движком (просто `{0} stacks`), а форма зафиксирована — берём нейтральную родительную: «Макс. {0} зарядов» / «Длительность {0} с» (число + сокращение, склонения избегаем).

**Правило безопасности:** не пытайся согласовать прилагательное с `{n}` — пол/число числа неизвестны заранее. Перестрой фразу так, чтобы плейсхолдер стоял в нейтральной позиции, либо используй plural-синтаксис.

## 6. Плейсхолдеры и разметка — НЕРУШИМО

### 6.1. Что нельзя трогать
- Числовые/именованные плейсхолдеры: `{0}`, `{1}`, `{amount}`, `{minValue}`, `{challengeName}`, `{stageNumber}`, `{currentReduction}` и т.п. — **копируются дословно**, включая фигурные скобки.
- Format-спеки: `{timeLimit:N0}`, `{finalScoreReward}` — двоеточие и спецификатор НЕ переводить.
- Теги разметки: `<1>…</1>`, `<2>…</2>`, `<t0>…</t0>`, `<b>…</b>`, `<color=#F2FF8D>…</color>`, `<sprite name="Gold">`, `<space=0.6em>` — **сохраняем парность и атрибуты буква-в-букву**. Переводится только текст МЕЖДУ тегами.
- Управляющие токены клавиш: `[W0]`, `[{4}s]` — внутри `[…]` это либо хоткей, либо вычисляемое значение; не переводим содержимое токенов.

### 6.2. Что можно
- Менять **порядок слов** вокруг плейсхолдеров (русский порядок свободнее английского) — при условии, что все плейсхолдеры присутствуют и теги остались парными.
  EN: `deals {0} damage` → RU: `наносит {0} урона`.
  EN: `{0} damage to Elites and Bosses` → RU: `{0} урона элитам и боссам`.
- Добавлять предлоги/падежные окончания вокруг плейсхолдера.

### 6.3. Контроль качества
Перед сдачей блока сверь: количество `{` = количество `}`; набор плейсхолдеров в RU = набор в EN; каждый открывающий тег имеет закрывающий; атрибуты `color`/`sprite name` не изменены. Любое расхождение = баг локализации.

## 7. Связность NAME ↔ DESC и кросс-доменная сверка

- Имя способности и все упоминания этой способности в DESC других морфов/трейтов/реликвий — **идентичны**. Если `Frost Orb` → «Ледяная сфера», то в описании Cryo Nova `{0} shatters…` подставляется та же «Ледяная сфера».
- Названия статусов (Bleed, Burn, Chill…) и характеристик одинаковы везде: в тултипе стата, в описании способности, в боевом логе, в Вечном Древе.
- Названия систем (Обелиск, Разлом, Перегрузка, Хаос, Вечное Древо) — строго по глоссарию во всех доменах.

## 8. Частые ловушки

- **«Overcharge»** — это игровая механика «Перегрузка» (ультимативная способность), НЕ перегрузка системы. Всегда с заглавной как термин.
- **«Chaos / Chaos Shard»** — внутриигровая валюта/ресурс «Хаос». Не «беспорядок».
- **«Major/Minor Affix»**, **«Major node»** — «крупный/малый». «Major Affix» → «Крупный аффикс».
- **«Relic» vs «Sigil» vs «Runestone»** — три разные сущности (Реликвия / Сигил / Рунный камень), не смешивать.
- **«Morph»** — эволюция способности, «Морф» (устоялось) либо «Превращение»; принято **«Морф»** (короче, влезает в UI).
- **«Salvage» vs «Scrap»** — действие «Разобрать» vs ресурс «Лом».
- **«Life» vs «Health»** — игра использует «Life» → «Здоровье» (привычнее РУ-игроку, чем «жизнь»); «Max Life» → «Макс. здоровье».
- **«Stash» vs «Inventory»** — «Хранилище» (в лагере) vs «Инвентарь» (на герое).
- Не калькировать имена предметов: «Worldbreaker Cuirass» → «Кираса сокрушителя миров», а не «Кираса Уорлдбрейкер».
